---
description: 【階段一】引導學生描述專案目標，由 Agent 撰寫並建立 requirements.md 文件，並以問答對齊細節
---

1. 檢查與讀取既有需求文件
   Agent 在啟動時，應主動檢查專案根目錄下是否已存在 `requirements.md` 或其他相關的需求描述文件。

2. 獲取或確認專案描述
   - **情境 A（無既有需求文件）**：請學生提供他們的開發構想、專案目標或問題描述。如果學生沒有提供足夠資訊，請主動詢問引導。
   - **情境 B（已有既有需求文件）**：Agent 主動讀取該檔案，向學生摘要已知的需求（包括專案目標、功能性需求與技術框架），並告知：「我看到你已經建立了需求文件，其中的核心目標是...，為了更具體規劃，我想確認以下幾點...」。

3. 撰寫或更新需求文件 (requirements.md)
   根據學生的描述或確認結果，建立或在既有檔案的基礎上更新專案根目錄的 `requirements.md` 檔案，內容應包含：
   - 專案目標與背景 (Project Goal)
   - 功能性需求 (Functional Requirements)
   - 技術要求 (Technical Requirements)：包含所採用的程式語言、技術棧或框架。
   - 非功能性需求（如環境與限制）(Non-functional Requirements)
   - 使用者情境 (User Scenarios)
   - 若學生對以上無法回答，則採取 AI Agent 的專業建議。

4. 需求對齊 (Alignment) 與細節澄清
   - 若為 **情境 A**：對學生提出 2~3 個具體的澄清問題，引導學生思考細節，並根據回覆更新 `requirements.md`。
   - 若為 **情境 B**：針對已有文件中模糊、不確定或需補充的部分提出 2~3 個澄清問題，進行對齊與微調。

5. 學生審查與確認
   引導學生審查 `requirements.md`。確認無誤後，提醒學生輸入 `/edu-plan` 進入下一個實作計畫階段。
