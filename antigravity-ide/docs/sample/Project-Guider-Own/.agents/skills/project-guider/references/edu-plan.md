---

1. 系統與架構設計
   讀取 `requirements.md`，分析所需要的檔案結構、程式邏輯流程、依賴庫，以及資料庫/資料結構設計。

2. 撰寫實作計畫 (Plan)
   在專案根目錄建立 `implementation_plan.md`。內容應包含：
   - 預計建立或修改的檔案與資料夾結構
   - 預計引入的套件或工具 (引導使用 uv 管理)
   - 詳細的實作步驟說明，主動解釋關鍵設計決策與架構選擇（例如：「為了維持模組化，我們會將資料處理與 GUI 分離」）
   - 驗證計畫（說明如何測試程式以驗證功能）
   > [!IMPORTANT]
   > 計畫完成後，請停止並等待學生審查與批准。在學生表示同意或點擊 Proceed 之前，不得擅自修改任何原始碼或執行破壞性指令。

3. 建立進度追蹤表 (Task)
   計畫被學生批准後，建立並初始化專案根目錄下的 `task.md`，將實作步驟拆解成具體可勾選的 TODO 列表（例如 `[ ]`, `[/]`, `[x]`）。

4. 開始實作開發 (Execution)
   按照 `task.md` 的步驟開始編寫與實作程式碼：
   - 所有的代碼均由 Agent 實作撰寫。
   - 每完成一個大步驟，須向學生簡要解釋實作內容、程式碼架構與邏輯。
   - 若遇到耗時的任務（例如下載大檔案、搜尋文獻等），可搭配使用 subagents 或 background tasks 進行平行執行。
   - 完成所有開發後，提醒學生輸入 `/edu-verify` 進入驗證與進化階段。
