# 成果報告 (walkthrough.md) - 車票合併作業 (Ticket Merger)

本專案已順利實作完成，我們成功建立了一套自動化車票處理工具，能夠解析高鐵電子車票、自動重命名與歸檔，並將其合併為附有總金額的 PDF 檔。

---

## 🛠️ 修改與實作摘要

我們在此專案中新增了兩個核心 Python 檔案，維持了模組化設計：

1.  **[pdf_parser.py](file:///home/dengkai/projects/google-ai-tools/antigravity-ide/Project-Guider/pdf_parser.py)**：
    *   負責讀取 `Tickets/` 下的 PDF 檔案。
    *   使用 `pdfplumber` 載入 PDF 文字。
    *   利用正則表達式，自動辨識搭乘日期（格式為 `YYYYMMDD`）、搭乘時間（格式為 `HHMM`）與票價。
    *   修正了票價正則表達式為 `\d+(?:,\d{3})*`，確保不含逗號的 4 位數票價（例如 `1160`）能被完整擷取，不會被截斷為 `116`。
2.  **[process_tickets.py](file:///home/dengkai/projects/google-ai-tools/antigravity-ide/Project-Guider/process_tickets.py)**：
    *   主流程腳本。
    *   自動在根目錄下尋找 `Tickets/` 並建立歸檔資料夾 `Tickets-[current_timestamp]`。
    *   將車票檔案複製並重命名為 `[日期][時間][票價]原檔名.pdf` 格式放入歸檔資料夾。
    *   將所有車票 PDF 合併，計算總價（本例為 `2,320`）。
    *   利用 `reportlab` 動態生成一個透明的 PDF 頁面，並使用 `pypdf` 的 `page.merge_page()` 將總價字樣 `Total Amount: NT$ 2,320` 疊加在合併 PDF 的最後一頁底部。
    *   輸出最終合併檔案 `Tickets-[執行日期]-[執行時間]-[總金額]-merge.pdf`。

## 💡 沉澱與學習知識點 (Evolution)

在這次的開發中，你可以學到兩個非常實用的 Python 技巧：

1.  **PDF 頁面疊加技術 (Page Overlay)**：
    *   通常我們要在 PDF 上面加水印或寫字時，如果直接修改 PDF 文字結構非常困難且容易跑版。
    *   **我們的做法**：使用 `reportlab` 繪製一個跟原本 PDF 尺寸一模一樣的「透明 PDF 頁面」，只在我們要的位置上畫字，然後用 `pypdf` 的 `page.merge_page()` 把它像貼貼紙一樣疊在原本的頁面上。這是一個非常強大、安全且不破壞原有排版的方式。
2.  **正規表達式的貪婪匹配與限制**：
    *   原本擷取票價時，正則運算式使用 `\d{1,3}` 來限制三位數以下。但對於不帶逗號的 `1160`，它只會配到前三位 `116`，導致嚴重的金額計算錯誤。
    *   修改為 `\d+` 讓它能貪婪地吃掉後面所有的連續數字，再以 `(?:,\d{3})*` 支援千分位逗號，成功解決了 Bug，這也體現了在寫正則表達式時，需多加考慮「不含分隔符號的連續數字」的邊界狀況。
